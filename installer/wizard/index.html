Recommended next steps

  1. Configure TLS — place your TLS certificate paths in config.yaml under server.tls_cert_file and server.tls_key_file, then restart the service.
  2. Store your ENCRYPTION_KEY in a secret manager (e.g. AWS Secrets Manager, HashiCorp Vault, or Azure Key Vault). If this key is lost, encrypted health data is unrecoverable.
  3. Invite clinical staff — use the admin dashboard to create practitioner accounts. Validate each practitioner's HPI CPN and APC before granting clinical access.
  4. Configure backups — ensure your PostgreSQL host has automated, encrypted backups retained for at least 7 years (HIPC Rule 6 / audit requirements).
  5. Review the audit log — access /api/v1/admin/audit to verify that audit events are being recorded correctly.
  6. Test NHI integration — perform a test $match operation against the NHI sandbox environment before going live.
